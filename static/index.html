DOCUMENTACION

Aviso!

Antes hay que ejecutar los siguentes 2 endpoint para cargar las habitaciones y los metodos de pago por defecto

POST

/metodos

-> Carga 4 metodos de pago, Tarjeta de credito, Tarjeta de debito, ejectivo y transferencia

POST

/habitaciones

-> Carga una lista de habitaciones por defecto, si no tiene la opcion de cargar uno por uno la que quiera

CLIENTES

GET

/clientes

-> Obtiene todos los clientes

GET

/clientes/:id

-> Encuentra el cliente que corresponda al ID pasado por parametro

POST

/clientes

-> Agrega un nuevo cliente

PUT

/clientes/:id

-> Actualiza el cliente que corresponda al ID pasado por parametro

RESERVAS

GET

/reservas

-> Obtiene todas las reservas

GET

/reservas/:id

-> Encuentra la reserva que corresponda al ID pasado por parametro

GET

/reservas?status=status

-> Filtra todas las reservas segun el estado que se pase.

POST

/reservas

-> Agrega una nueva reserva, con los datos del cliente, el cuarto y los dias de estadia

PUT

/reservas/:id?status=status

-> Actualiza el estado del la reserva, solamente se pueden pasar 3 estados, Pendiente, Pagado o Eliminado.

DELETE

/reservas/:id

-> Elimina la reserva por su ID

PAGOS

GET

/pagos

-> Obtiene todas los pagos

GET

/pagos/:id

-> Encuentra el pago que corresponda al ID pasado por parametro

PUT

/pagos/:id

-> Actualiza el pago

DELETE

/pagos/:id

-> Elimina el pago por medio de su ID

HABITACIONES

GET

/habitaciones

-> Obtiene todas las habitaciones

GET

/habitaciones/:id

-> Encuentra la habitacion que corresponda al ID pasado por parametro

METODOS DE PAGOS

GET

/metodos

-> Obtiene todos los medios de pago

GET

/metodos/:id

-> Encuentra el metodo de pago que corresponda al ID pasado por parametro
